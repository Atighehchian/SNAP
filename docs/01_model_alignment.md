# Model alignment

This generator is aligned with the SNAP mathematical model parameter structure.

## Fixed time data

The time structure is not generated randomly:

```text
S = 2 shifts
T = 6 days per week
K = 16 weeks
```

## Independent parameters

The generator creates these model inputs directly:

| Parameter | Meaning |
|---|---|
| `B(w,h)` | Ward `w` exists in hospital `h` |
| `Du(d)` | Required total clinical training days for course `d` |
| `O(d)` | Consecutive training days per week for course `d`, between 2 and 6 |
| `Mea(d,w)` | Course `d` can be taught in ward `w` |
| `Comm(i,d)` | Group `i` requires course `d`; generated by semester curriculum |
| `PW(d)` | If 1, one eligible ward is selected; if 0, all defined wards must be visited |

## Dependent parameters

The generator derives:

| Parameter | Rule |
|---|---|
| `Dur(d,w)` | For `PW=1`, each eligible ward can cover full `Du(d)`; for `PW=0`, `Du(d)` is split across all defined wards so the positive `Dur(d,w)` values sum to `Du(d)` |
| `weeks(d,w)` | `ceil(Dur(d,w) / O(d))` |
| `Com(i,d,w)` | Derived as `Comm(i,d) × Mea(d,w)` for all `PW`; the MIP uses it mainly in `PW=0` constraints |
| `Cap(d,w,h)` | Emitted for every `d,w,h`; invalid combinations receive zero, valid combinations are raised when needed to make the planted feasible schedule capacity-valid |
| `Av(i,t,k)` | Semester-based: all groups in the same semester share the same available days across all weeks |
| `Avb(w,h,t,k)` | Usually set to available to avoid artificial infeasibility |
| `ne(d,w)` | Long-shift permission; zero in small profiles, limited in larger profiles |
| `nt(w)` | Interference permission; zero in small/MIP profiles, limited in hard profiles |
| `nee(d,w)` | Set to zero for the current study |
| `Eb(d,w,h)` | Derived from `Mea(d,w)` and `B(w,h)` |


## Semester curriculum policy

Version 1.4.1 enforces the curriculum logic used by the manuscript author:

- Each course belongs to exactly one semester.
- All groups in the same semester require the same courses.
- Courses are not shared between junior and senior cohorts.
- The weekly availability matrix `Av(i,t,k)` is shared by all groups of the same semester.
- `Du(d)` is generated from realistic values and is never below 6 days.
- Small misleading fields such as `level`, `allowed_levels`, and `allowed_groups` are not emitted. If junior/senior logic is needed internally, it is derived from the semester number.

## Planted feasible solution

Each generated instance includes a known feasible schedule. This is used to confirm that the generated instance is not infeasible by construction.


## Schema clarity note

The set `I` is represented as student groups. Generated files use `group_id` only. Fields such as `student_id`, duplicate `group`, and legacy compatibility views are intentionally omitted to avoid misleading readers.


## Capacity generation refinement in v1.4.3
For every valid `(d,w,h)` where `Eb(d,w,h)=1`, `Cap(d,w,h)` is generated independently and demand-aware. If a course is defined in ward `w` and multiple hospitals contain that ward, all corresponding hospitals receive meaningful positive capacities. Capacities are not forced to be equal across hospitals, although equality may occur naturally. Invalid combinations keep `Cap(d,w,h)=0`.


## v1.4.5 very small profiles

Two exact-test size profiles were added: `vvs` with I=3, D=1, W=1, H=1 and `vs` with I=4, D=2, W=2, H=1. They are intended for very small mathematical-model tests and early solver validation.
